# 要件定義

## 機能要件

### 1. KIF 読み込み

- KIF ファイルを受け付ける（解析なし・水匠解析付きの両方）
- UTF-8 / Shift-JIS（CP932）の KIF をユーザーの事前変換なしで受け付ける
- サーバー上の指定フォルダから `.kif` ファイルを一括取り込みできる
- 以下を抽出する
  - 対局情報（対局日時・先後手・勝敗・手数）
  - 指し手リスト
  - 評価値・読み筋・候補手（水匠解析付きの場合のみ）
- 同一対局の重複取り込みを検出する（MVP は raw KIF の一致で判定し、将来的にハッシュや対局日時・先後手の補助判定を検討する）

### 2. SFEN 生成

- KIF 手を USI に変換する
- USI を盤面へ適用する
- KIF を再生して各手番の局面を SFEN 文字列で生成・保存する
- 初手（開始局面）から終局まで全手番分を保存する
- SFEN は戦法判定・囲い判定・定跡 DB・AI 解説の共通キーとして扱う

### 3. 評価値保存

- 手番ごとに以下を保存する（解析なし KIF の場合は全て NULL）

| フィールド   | 型      | NULL 許容 | 内容                         |
| ------------ | ------- | --------- | ---------------------------- |
| `eval`       | integer | ✅        | 先手視点の評価値（cp）       |
| `best_move`  | string  | ✅        | 水匠が示す最善手（USI 形式） |
| `pv`         | string  | ✅        | 読み筋（スペース区切り USI） |
| `candidates` | JSON    | ✅        | 候補手リスト（手＋評価値）   |

### 4. 棋譜閲覧

- 対局一覧を表示する（対局日時・先後・勝敗・手数）
- 対局を選択すると盤面ビューアを開く
- 盤面は手番を進める／戻すことができる

### 5. グラフ表示・局面連動

- 評価値がある対局は評価値の推移を折れ線グラフで表示する（先手視点）
- 評価値がない対局はグラフ非表示（手数バーのみ表示）
- グラフ上の点をクリックすると対応する手番の局面へ移動する

### 6. 局面解説

- 選択した局面を LLM で解説するため、局面情報を同じ入力形式へ正規化する
- 解説材料には以下を含める
  - SFEN
  - 手数
  - 実戦手
  - 評価値
  - 最善手
  - 読み筋
  - 候補手
  - 自分専用定跡候補
- 解析付き KIF 由来の評価値と水匠後解析由来の評価値を区別せず、同じ `positions` の値として扱う
- 評価値・候補手・読み筋がない局面では、不足項目を明示してプロンプトを生成する
- 初期実装では LLM 呼び出しを行わず、LLM に渡すプロンプトを API で返す

## 非機能要件

| 項目         | 目標                                      |
| ------------ | ----------------------------------------- |
| 対応ブラウザ | Chrome 最新版（個人利用のため最小限）     |
| 取り込み速度 | KIF 1 ファイル（200 手）を 1 秒以内に処理 |
| DB サイズ    | 棋譜 1,000 局・局面 200,000 件程度を想定  |
| 認証         | なし（ローカル or 個人サーバー運用）      |

## スコープ外（MVP）

- マルチユーザー対応
- モバイル最適化
- リアルタイム解析（水匠との直接通信）
- 棋譜の編集・コメント入力
- 外部定跡 DB の取り込み
- 水匠 USI エンジンの直接起動・後解析
- AI 解説生成

## 将来要件

### 水匠解析連携

- 水匠5を USI エンジンとして外部プロセス起動する
- 保存済み `positions.sfen` を `position sfen ...` で水匠へ渡す
- `go depth N` で解析を実行する
- USI 出力から以下を取得する
  - 評価値 `eval`
  - 最善手 `best_move`
  - 読み筋 `pv`
  - 候補手 `candidates`
- 取得した解析結果を `positions` へ保存する
- 局面単位の解析 API を用意する
- 対局単位の一括解析 API は将来拡張とする

API:

| メソッド | パス | 内容 |
|---|---|---|
| `POST` | `/api/positions/{id}/analyze` | 指定局面を水匠で解析する |
| `GET` | `/api/positions/{id}/explanation-prompt` | 指定局面の解説用プロンプトを返す |
| `POST` | `/api/positions/{id}/explain` | 指定局面の解説を外部 LLM コマンドで生成する |
| `GET` | `/api/blunders/explanation-prompt?game_id={id}&move_number={n}` | 悪手理由解説用プロンプトを返す |
| `POST` | `/api/blunders/explain` | 悪手理由解説を外部 LLM コマンドで生成する |

局面解説の生成では、`llm_command` または `SHOGI_DB_LLM_COMMAND` で指定した外部コマンドへプロンプトを標準入力で渡し、標準出力を解説文として受け取る。shogi-db 本体は特定の LLM サービスや SDK に依存しない。

解説プロンプトは、SFEN・実戦手・評価値・最善手・候補手・読み筋・自分専用定跡候補を構造化して渡す。LLM には、確定情報と推測を分けること、根拠に使った入力項目を明示すること、不足項目がある場合は断定しないことを求める。

悪手理由解説では、悪手ランキングの `game_id` と `move_number` を指定し、着手前 SFEN、着手後 SFEN、実戦手、前後評価値、評価値低下量、候補手、読み筋を入力材料にする。読み筋や候補手が不足している場合は、プロンプト上で不足を明示する。
生成 API では、局面解説と同じ `llm_command` または `SHOGI_DB_LLM_COMMAND` を使い、悪手理由解説用プロンプトを標準入力で渡す。

将来 API:

| メソッド | パス | 内容 |
|---|---|---|
| `POST` | `/api/games/{id}/analyze` | 指定対局の全局面または未解析局面を水匠で解析する |

水匠は shogi-db に組み込まず、外部プロセスとして扱う。
