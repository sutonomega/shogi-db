# プロジェクト概要

## 目的

自分の棋譜を蓄積・分析し、以下を実現する個人用将棋データベースを構築する。

- **苦手局面の発見** — 評価値が急落する局面・手を把握する
- **戦法分析** — よく指す戦法・囲いの勝率を集計する
- **定跡調査** — 自分の実戦から生成した定跡 DB を参照する
- **水匠解析の活用** — 評価値・読み筋・候補手をそのまま保存して後から閲覧する

## 基本方針

shogi-db は SFEN を中心にした局面データベースとして設計する。

```
KIF
↓
KIF パーサー
↓
USI
↓
盤面再生
↓
SFEN
↓
DB 保存
```

盤面の正規表現として SFEN を採用することで、戦法分析・囲い分析・定跡 DB・AI 解説まで一貫したデータ構造で扱う。

## 対象ユーザー

自分（個人利用）。マルチユーザー対応は当面スコープ外。

## 対応入力フォーマット

| フォーマット        | 内容                                     |
| ------------------- | ---------------------------------------- |
| KIF（解析なし）     | 対局情報・指し手のみ                     |
| KIF（水匠解析付き） | 評価値・読み筋・候補手が埋め込まれたもの |

評価値は「あれば使う、なければ NULL」として扱う。評価値がない棋譜でも対局一覧・盤面表示・局面移動は正常に動作する。

## MVP スコープ

「棋譜ビューア＋評価値グラフ」を最初のゴールとする。

```
1. KIF 読み込み
2. SFEN 生成
3. 評価値保存（あれば）
4. 対局一覧
5. 盤面表示
6. 評価値グラフ（評価値がある対局のみ表示）
7. グラフクリックで局面移動
```

## 将来的な機能

- 戦法・囲いの自動タグ付けと集計
- 悪手ランキング（評価値急落）
- 定跡 DB 生成（自分の実戦から）
- 外部定跡 DB 連携（やねうら王 DB などを SFEN 列へ変換して取り込む）
- 自分・外部定跡・プロ棋譜・AI 最善手の比較
- AI による局面解説

## 開発方針

### 定跡 DB フォーマット

SBK 形式は内部形式としては使用しない。SFEN をキーにした独自 DB で管理する。

理由:

- 自由度が高い
- 評価値・候補手を一緒に保存できる
- source を持たせることで自分の実戦、外部定跡、プロ棋譜などを同時管理できる
- 必要に応じて外部形式への変換も可能

外部定跡 DB は最終的に局面列へ変換できれば取り込める。HiraganaSuisho や BookConv などの公開ツールを利用し、やねうら王 DB などを SFEN ベースの shogi-db 形式へ変換できる可能性がある。

### 戦法・囲い判定

独自実装で判定する。局面の駒配置（SFEN）をもとにルールベースで戦法・囲いを判定する。

### DB

MVP は SQLite で構築し、必要に応じて PostgreSQL へ移行する。
