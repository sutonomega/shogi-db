# 開発ロードマップ

Issue の元ネタドキュメント。各タスクは GitHub Issue として切り出す。

---

## 基本思想

shogi-db は SFEN を中心にした将棋データベースとして設計する。

```
KIF
↓
KIF パーサー
↓
USI
↓
盤面再生
↓
SFEN
↓
DB 保存
```

SFEN を局面の共通キーにすることで、棋譜ビューア、戦法・囲い分析、悪手分析、定跡 DB、外部定跡 DB 連携、AI 解説を同じデータ構造で扱う。

## Phase 1 — MVP「棋譜ビューア＋評価値グラフ」

まず「棋譜を見て評価値を確認できる」状態を作る。

### バックエンド

- [x] **KIF パーサー実装** (#1)
  - KIF 形式（標準・棋桜）のヘッダー解析（対局日時・先後手・勝敗）
  - 指し手リスト解析
  - 水匠解析付き KIF から評価値・読み筋・候補手を抽出

- [x] **KIF 文字コード自動判定対応** (#35)
  - UTF-8 / Shift-JIS（CP932）の KIF を事前変換なしで取り込む
  - API 取り込み時に KIF bytes を自動デコードする

- [x] **ブラウザからの KIF ファイル取込** (#40)
  - 対局一覧画面から `.kif` ファイルを選択して取り込む
  - 取り込み成功時に対局一覧・集計・悪手ランキングを更新する
  - 取り込み失敗時にエラーを表示する

- [x] **SFEN 生成** (#2)
  - KIF 手を USI へ変換
  - USI を盤面へ適用
  - KIF を手番ごとに再生して SFEN 文字列を生成
  - 将来的に `python-shogi` 互換の Move / USI 連携を検討

- [x] **DB 保存** (#3)
  - `games` テーブルへの対局情報保存
  - `positions` テーブルへの全手番保存（SFEN・評価値・候補手）

- [x] **API 実装** (#4)
  - `POST /api/games/import` — KIF 取り込み
  - `GET /api/games` — 対局一覧
  - `GET /api/games/{id}/positions` — 全局面データ

### フロントエンド

- [x] **対局一覧画面** (#5)
  - 対局日時・先後手・勝敗・手数の一覧表示
  - 対局をクリックしてビューアへ遷移

- [x] **盤面表示** (#6)
  - SFEN から盤面を描画
  - 駒の配置・持ち駒の表示

- [x] **評価値グラフ** (#7)
  - 先手視点の評価値推移を折れ線グラフで表示
  - 手番を横軸、評価値を縦軸

- [x] **グラフ連動局面移動** (#8)
  - グラフの点をクリックしたとき対応する手番の局面へジャンプ
  - 手番スライダー or 前後ボタンでの移動

## Phase 2 — 分析機能

棋譜ビューアが完成したら戦法・悪手分析を追加する。

- [x] **戦法判定** (#9)
  - 局面の駒配置（SFEN）をもとにルールベースで独自実装
  - 例: 角換わり、横歩取り、相掛かり、四間飛車
  - 判定結果を `games.strategy` へ保存

- [x] **囲い判定** (#10)
  - 玉・金銀の配置パターンをもとにルールベースで独自実装
  - 例: エルモ囲い、居飛車穴熊、美濃囲い、銀冠
  - 判定結果を `games.enclosure` へ保存

- [x] **戦法別集計** (#11)
  - 戦法ごとの対局数・勝率を集計して表示
  - 例: 角換わり 123 局、勝率 48%

- [x] **囲い別集計** (#12)
  - 囲いごとの対局数・勝率を集計して表示
  - 例: エルモ囲い 56 局、勝率 55%

- [x] **悪手分析（評価値急落ランキング）** (#13)
  - 評価値の急落（Δeval が大きい手）を抽出してランキング表示
  - 例: 37 手目 ▲4 六角、平均 −120

## Phase 3 — 定跡 DB

- [ ] **SFEN 集計バッチ** (#14)
  - `positions` テーブルの全レコードを集計
  - 同一 SFEN ごとに指し手の出現回数・平均評価値を計算
  - 自分の実戦由来データとして `source=self` を付与

- [ ] **指し手頻度分析** (#15)
  - 局面ごとの最頻手・評価値の分布を可視化
  - SFEN ごとに「よく指した手」「出現回数」「平均評価値」を確認

- [ ] **定跡 DB 生成** (#16)
  - 集計結果を `openings` テーブルへ UPSERT
  - `openings.source` で self / yaneou / professional / floodgate などの由来を分ける
  - 外部定跡 DB は SFEN 列へ変換できる場合に取り込み可能にする

- [ ] **自分専用定跡表示** (#17)
  - 盤面ビューア上で「この局面でよく指した手」を重ねて表示
  - 例: 角換わり → 早繰り銀 → この局面での頻度順候補手

## Phase 4 — AI / エンジン活用

- [ ] **水匠解析連携** (#27)
  - 保存済み SFEN を水匠5へ `position sfen ...` で渡す
  - `go depth N` の USI 出力から評価値・最善手・読み筋・候補手を取得
  - `positions` に `eval` / `best_move` / `pv` / `candidates` / `analyzed_at` / `engine_name` / `engine_depth` を保存
  - `POST /api/positions/{id}/analyze` と `POST /api/games/{id}/analyze` を検討
  - MVP では実装せず、解析付き KIF がない棋譜を後から補完する機能として扱う

- [ ] **局面解説** (#18)
  - 選択した局面について LLM に「なぜ評価値が下がったか」を解説させる
  - SFEN、実戦手、評価値、候補手、読み筋を入力にする
  - 水匠解析連携で後から得た評価値も利用できるようにする

- [ ] **悪手理由解説** (#19)
  - 悪手ランキングの各手について自然言語で解説を生成
  - 評価値急落局面の SFEN と候補手差分を利用する
  - 解析付き KIF がない棋譜は水匠解析後の評価値を利用する

- [ ] **定跡比較** (#20)
  - 同じ SFEN に対して自分の実戦手、自分専用定跡、外部定跡 DB、プロ棋譜、AI 最善手を比較
  - 水匠の最善手・評価値とも比較する
  - 比較結果の差異を自然言語で解説

## 最終目標

単なる棋譜ビューアではなく、棋譜 DB、局面 DB、自分専用定跡、外部定跡 DB 連携、AI 解説まで発展させる。

最終的には「自分専用の将棋研究環境」を作る。
