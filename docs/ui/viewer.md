## コンポーネント仕様

### KIF 登録

対局一覧画面の棋譜登録エリアで、自分の実戦棋譜を `games` / `positions` に保存する。定跡登録と同じ管理エリアの並びに置き、用途の違いが分かるようにする。

| 項目         | 仕様                                               |
| ------------ | -------------------------------------------------- |
| 入力         | `.kif` ファイル選択                                |
| 文字コード   | UTF-8 / Shift-JIS（CP932）をバックエンドで自動判定 |
| 送信         | `POST /api/games/import` にファイル bytes を送る   |
| 一括登録     | `POST /api/games/import-directory` に `async=true` でフォルダパスを送り、進捗とキャンセルを表示する |
| 定跡DB更新   | `POST /api/openings/rebuild` に `async=true` を送り、自分の棋譜由来の定跡 DB 更新進捗とキャンセルを表示する |
| 配置         | 単体登録と一括登録を同じ行に並べ、定跡登録と同じ操作レイアウトにする |
| 成功時       | 対局一覧・集計・悪手ランキングを再読み込みする     |
| 失敗時       | 画面上部のステータス領域にエラーを表示する         |

### 定跡登録

対局一覧画面の定跡登録エリアで、定跡として扱う KIF を `openings(source=professional)` に登録する。自分の実戦棋譜は KIF 登録で `games` / `positions` に保存し、指し手頻度の集計元として扱う。UI に表示する説明は「定跡として登録する棋譜」とし、内部 source 名の `professional` は表示しない。

| 項目       | 仕様                                                    |
| ---------- | ------------------------------------------------------- |
| 単体登録   | `POST /api/openings/import?source=professional` に KIF bytes を送る |
| 一括登録   | `POST /api/openings/import-directory` に `async=true` でフォルダパスを送り、`GET /api/openings/import-directory/jobs/{id}` で進捗を表示する |
| 配置       | 単体登録と一括登録を同じ行に並べ、棋譜登録と同じ操作レイアウトにする |
| 外部定跡   | 将来 `source` を選ぶ登録 UI を追加できる構成にする       |
| キャンセル | 一括登録中はキャンセルボタンを表示し、`POST /api/openings/import-directory/jobs/{id}/cancel` で停止要求する |
| 失敗時     | 画面上部のステータス領域にエラーを表示する              |

### 盤面（Board）

| 項目       | 仕様                                    |
| ---------- | --------------------------------------- |
| 描画方式   | SVG または Canvas                       |
| SFEN 入力  | `positions[moveNumber].sfen`            |
| 向き       | 先手（下）・後手（上）固定              |
| 持ち駒     | 盤面の上下に表示                        |
| ハイライト | 直前の指し手（from / to）を色付きで強調 |
| 最善手表示 | Phase 2 以降: 矢印オーバーレイ（任意）  |

### 評価値グラフ（EvalGraph）

| 項目            | 仕様                                               |
| --------------- | -------------------------------------------------- |
| ライブラリ      | Recharts（`LineChart`）                            |
| X 軸            | 手数（0 〜 move_count）                            |
| Y 軸            | 評価値（先手視点 cp）、通常表示は ±3000 にクリップ |
| 基準線          | Y=0 に破線                                         |
| 現在位置        | 縦線（`ReferenceLine`）で現在手番を強調            |
| クリック        | グラフの点クリック → `setMoveNumber(index)`        |
| 評価値が全 NULL | グラフ自体を非表示にし「解析データなし」と表示する |
| 一部手番が NULL | NULL の点を省略してラインを繋がずギャップ表示      |
| 大評価値        | `+32000` などの範囲外評価値は上下端に表示し、詳細値はツールチップで確認する |
| 詰み評価値      | `+100000` / `-100000` は「詰み」として別表示し、通常スケールを広げない |

### 局面情報パネル（PositionPanel）

表示内容:

| 項目   | 内容                           |
| ------ | ------------------------------ |
| 手番   | `{move_number}手目`            |
| 指し手 | USI から変換した日本語表記（例: ▲７六歩、△同銀、▲５五角打） |
| 評価値 | `+320 → -80 (Δ-400)` ※前手比   |
| 最善手 | 水匠の最善手と評価値           |
| 候補手 | 上位 3〜5 手を評価値付きで表示 |
| 読み筋 | KIF 表記に変換して表示         |

盤面は現在表示中の指し手に対応する移動元・移動先をハイライトする。駒打ちは移動先のみをハイライトし、開始局面ではハイライトしない。評価値グラフ、手番ボタン、スライダーのいずれで局面移動してもハイライトを更新する。
指し手頻度と定跡の候補手も、現在局面の SFEN をもとに同じ日本語表記へ変換して表示する。

エンジン候補手は現在局面の `positions[].candidates` から上位 5 件を表示する。各候補手は日本語表記と評価値を並べ、現在局面の実戦手と一致する候補手を区別する。候補手を選択すると、その候補手の移動元・移動先を盤面に反映し、手番移動時には選択をクリアする。
候補手が未保存の局面では空表示になり、水匠解析付き KIF を登録するか「この局面を解析」ボタンで `POST /api/positions/{id}/analyze` を実行すると表示される。解析ボタンはサーバー側の `SUISHO_ENGINE_PATH` を使い、未設定の場合は API エラーを画面上部に表示する。

### 局面解説

Phase 4 の UI として、現在表示中の局面について解説用プロンプトと生成結果を確認できる。

| 項目     | 仕様                                                             |
| -------- | ---------------------------------------------------------------- |
| 操作     | 「プロンプト作成」ボタンで `GET /api/positions/{id}/explanation-prompt` を呼ぶ |
| 生成     | 「解説生成」ボタンで `POST /api/positions/{id}/explain` を呼ぶ   |
| 表示     | API が返す `prompt` と `explanation` を等幅テキストで表示する    |
| LLM設定  | サーバー側の `SHOGI_DB_LLM_COMMAND` に外部 LLM コマンドを設定する |
| 不足項目 | API が返す `materials.missing` を「不足: ...」として表示する     |
| 手番移動 | 別手番へ移動したら表示中プロンプトと解説結果をクリアする         |
| 失敗時   | 画面上部のステータス領域にエラーを表示する                       |

### 定跡比較

Phase 4 の UI として、現在表示中の局面について自分の実戦頻度手、source 別定跡候補、エンジン候補手の比較結果を確認できる。

| 項目     | 仕様                                                             |
| -------- | ---------------------------------------------------------------- |
| 操作     | 「比較作成」ボタンで `GET /api/positions/{id}/opening-comparison-prompt` を呼ぶ |
| 生成     | 「比較解説」ボタンで `POST /api/positions/{id}/opening-comparison-explain` を呼ぶ |
| source   | 初期 UI では `self,professional,yaneou` を比較対象にする         |
| 表示     | API が返す `prompt` と `explanation` を等幅テキストで表示する    |
| 不足項目 | API が返す `materials.missing` を「不足: ...」として表示する     |
| 手番移動 | 別手番へ移動したら表示中プロンプトと解説結果をクリアする         |
| 失敗時   | 画面上部のステータス領域にエラーを表示する                       |

### 悪手理由解説

対局一覧画面の悪手ランキングで、各悪手の理由解説を生成して確認できる。

| 項目     | 仕様                                                               |
| -------- | ------------------------------------------------------------------ |
| 操作     | 各悪手カードの「理由生成」ボタンで `POST /api/blunders/explain` を呼ぶ |
| 入力     | 悪手カードの `game_id` と `move_number` を送る                     |
| 表示     | 評価値損失の大きい手をランキング表示し、同じ局面キー（盤面・手番・持ち駒）で同じ指し手が出た発生回数も表示する |
| 指し手表記 | 着手前 SFEN を使い、USI ではなく盤面ビューアと同じ日本語表記で表示する |
| 局面移動 | 各悪手カードの「局面へ」ボタンで `/games/{game_id}?move={move_number}` へ遷移する |
| LLM設定  | サーバー側の `SHOGI_DB_LLM_COMMAND` に外部 LLM コマンドを設定する   |
| 失敗時   | 画面上部のステータス領域にエラーを表示する                         |

## 画面遷移

```
/                   トップ（対局一覧）
    ↓ 対局をクリック
/games/{id}         棋譜ビューア（初期手番 = 0）
    ↓ グラフ or ボタン操作
/games/{id}?move=37 特定手番へジャンプ（URL に手番を反映）
```

## 状態管理

```typescript
// ビューアの状態
interface ViewerState {
  game: Game; // 対局情報
  positions: Position[]; // 全手番の局面データ
  currentMove: number; // 現在表示中の手番
}

// 局面データ（1 手番分）
interface Position {
  moveNumber: number;
  sfen: string;
  move: string | null; // USI 形式
  eval: number | null;
  bestMove: string | null;
  pv: string | null;
  candidates: Candidate[];
}

interface Candidate {
  move: string; // USI 形式
  eval: number;
}
```
